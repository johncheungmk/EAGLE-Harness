Experiment

The current study is a preliminary expert-reviewed prototype evaluation for an IT helpdesk chatbot, not a full production benchmark.

Evaluation setup

CorpusStudent-facing IT helpdesk FAQ corpus covering Wi-Fi, Blackboard, printing, password management, equipment loan, email, SCC/IT service counters, CPCE App, Microsoft account, and ITU services.
Question set45 IT helpdesk questions across factual, troubleshooting, escalation, action, and adversarial categories.
SystemsStandard RAG, Agentic RAG, and EAGLE Harness.
Generated answers135 answers.
Human review2 reviewers, 270 answer-level reviews, six criteria rated from 1 to 5.

Systems compared

SystemDescription
Standard RAGRetrieval and answer generation only; no governance enforcement.
Agentic RAGPartial governance; high-risk outcomes are softened to confirmation required.
EAGLE HarnessEvidence-aware retrieval, policy/risk decision, access filtering, and evidence graph logging.

Human-review rubric

CriterionMeaning
CorrectnessWhether the answer is factually correct for the IT helpdesk question.
GroundednessWhether the answer is supported by retrieved evidence.
Citation precisionWhether the cited sources support the answer.
CompletenessWhether the answer covers the main user need.
UsefulnessWhether the answer would help a student resolve the issue.
Risk handlingWhether the system correctly allows, warns, escalates, or blocks risky requests.

Scope

The study uses TF-IDF retrieval and lightweight rule-based governance. It does not claim superiority over hybrid retrieval, GraphRAG, NeMo Guardrails, Guardrails AI, OPA/Casbin, or enterprise LLM gateway systems.
