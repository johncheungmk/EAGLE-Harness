Implementation Guide

This guide explains how to implement EAGLE as a small research prototype before moving to a production system.

Minimum viable implementation

  1. Collect institutional documents and assign metadata.
  2. Build the standard RAG baseline.
  3. Add agentic RAG behaviour with tool-style actions.
  4. Add EAGLE modules: evidence graph, authority ranking, conflict detection, policy engine, and risk manager.
  5. Run batch evaluation and human review.

Document metadata

Each document should include:

doc_id, title, topic, owner, version, effective_date,
authority, access_level, sensitivity, status, text

Evidence score

evidence_score =
  0.5 * retrieval_score
+ 0.3 * authority_score
+ 0.2 * freshness_score

This can be adjusted after pilot testing.

Policy decision levels

DecisionMeaning
allowThe answer or low-risk operation can proceed.
confirmation_requiredThe user should confirm before action.
human_approval_requiredA responsible human approver is needed.
blockThe answer or action should not be provided.
